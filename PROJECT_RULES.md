# Eureka Project Rules

## Product
Eureka is a conversational quant research engine for markets/macro/econ.
It turns vague ideas into structured hypotheses, finds and links datasets, runs real tests/backtests, and records the investigation in a transparent notebook with branching and user steering.

## Priorities
1. Best Overall / Most Promising Startup
2. Product credibility over gimmicks
3. Execution and workflow depth over breadth
4. Transparent provenance and notebook history
5. Quant rigor and anti-leakage
6. Clean light institutional UI

## Hard constraints
- Python-first backend/orchestration
- OpenAI Responses API
- GPT-5.4
- Structured outputs / typed schemas
- Human-in-the-loop workflow
- No dark gradient UI
- No fake hardcoded flows pretending to be dynamic

## v1 sources
- FRED
- market data source
- SEC EDGAR

## v1 analysis
- summary stats
- regression
- rolling regression
- event study
- simple backtest
- regime analysis

## Non-goals
- general domain support
- auth complexity
- multi-user collaboration
- unnecessary ML model training
